# 炉痕地牢 AI 交接文档

更新时间：2026-07-17

## 一句话状态

这是一个无框架、移动端优先、可直接部署到 GitHub Pages 的 PVE 三线卡牌地牢。当前主循环已经从 7 房间限时判分版，重构为 3 层 / 15 房间、击杀制、牌库循环和遗物构筑版。

## 用户已经确认的方向

- 删除自由铸卡与自定义起始牌，专注 PVE 地牢。
- 地图要有分岔、层级、精英、事件、营火、宝库和层主。
- 战斗只有击杀怪物才结束，不能再按 4/6 回合和场面分结算。
- 要有抽牌、弃牌、洗牌循环；阵亡单位也应回到弃牌堆。
- UI 不要像说明书，尽量使用短标签和图形反馈。
- 战斗需要背景，手牌采用类似炉石的扇形排布。
- 卡图、费用、攻击、生命、效果是最重要的信息；所有场景卡牌必须保持模板一致。

## 当前流程

首页 → 选择英雄 → 可预览 8 张初始牌 → 开始远征 → 三层分岔地图 → 房间 → 奖励/遗物 → 层主 → 下一层 → 最终首领。

每层固定 5 个深度：前 4 个深度通常有两个选项，第 5 个深度为层主。总深度 `TOTAL_FLOORS = 15`。

房间类型：

- `battle`：普通战，胜利后三选一卡。
- `elite`：强化敌人，胜利后三选一遗物。
- `event`：随机事件。
- `camp`：治疗、强化或删牌。
- `relic`：直接三选一遗物。
- `boss`：层主；前两层胜利后获得遗物，第三层胜利通关。

## 战斗规则

- 三条战线，玩家每回合最多打出一张牌，也可以空过。
- 单位选择后需要点空战线；已有友军的战线不能被覆盖。
- 敌方优先部署到空战线。
- 同线单位互相攻击，空线单位直击英雄。
- 仅当玩家或敌方英雄生命归零时结束战斗。
- 初始能量为 1，逐回合增长，通常最高 10；遗物可以改变它。
- 默认起手 4 张，每回合抽 1 张，手牌上限 8。
- 法术结算后进入弃牌堆。
- 玩家单位阵亡后，原卡进入弃牌堆。
- 牌库为空且需要抽牌时，弃牌堆洗回牌库。
- 战斗动画为双方单位向中线对撞、中心爆炸、单位或英雄显示伤害数字。

关键函数：`startBattle`、`drawCards`、`reshuffleDiscard`、`lockTurn`、`attack`、`endCheck`、`battleWin`。

## 遗物

当前共 10 件，定义在 `RELICS`：铸甲碎片、回响齿轮、余烬棱镜、火种核心、铁砧吊坠、血煤、贪婪之眼、尖牙徽记、灰烬瓶、钟摆。

遗物通过 `gainRelic()` 获取，`offerRelic()` 生成三选一。地图和战斗中均可查看已有遗物。

## 存档

- `forge_meta`：晶尘、通关次数、最佳深度。
- `forge_run`：英雄、生命、牌组、路线、当前深度、历史、遗物和晶尘。
- 新远征存档版本为 `version: 2`。旧版没有 `route` 的远征会自动清除，避免旧结构卡死新地图。

背景提示词摘要：原创暗黑奇幻、黑铁教堂、青灰雾与余烬橙、竖屏 2:3、中央留白、无人物/卡牌/UI/文字/水印。

`cardFace()` 是唯一卡牌组件。手牌、战场、奖励、图鉴、起始牌组预览和长按详情都应继续复用它，不要复制出另一套牌面。

## 移动端 UI

- `flow.css` 最后部分是 PVE 重构的高优先级覆盖。
- `#map-view` 和 `#battle-view` 使用本地背景图，不依赖远端 URL。
- 手牌角度和上下偏移由 `renderHand()` 写入 `--fan-angle`、`--fan-y`、`--fan-z`。
- 低高度屏幕由 `@media(max-height:720px)` 收紧卡牌和战场。
- 长按卡牌 380ms 打开放大详情和关键词解释。
